Janitor Dashboard — Architecture Approaches

How to structure the AI-powered janitor page

A

Tab-Based Dashboard

Single page with horizontal tabs: Overview | Dedup | Cleanup | AI Triage. Chat panel slides in from the right as a drawer. Each tab is a focused workflow.

+ Clean separation of concerns per tab
+ Familiar pattern (matches DataAPI storage-tool)
+ Each tab can load data independently
- Context switching between tabs
- AI chat is separate from the data you're looking at
B

Command Center (Recommended)

Vertically scrolling single page: health metrics at top, then dedup groups, cleanup suggestions, file triage — all visible. Persistent AI chat panel on the right side (collapsible). AI can highlight/annotate any section inline.

+ Everything visible at once — no context switching
+ AI chat sees what you see, can reference sections
+ Matches DocJanitor's single-page feel
- Longer page, more scrolling
- Need to be smart about lazy-loading sections
C

Split-Pane Workspace

Left pane: AI chat + command input (always visible). Right pane: dynamic content area that shows whatever the current task is — dedup results, file triage table, cleanup preview. Chat drives what the right pane shows.

+ AI-first experience — chat is the primary interface
+ Dynamic, feels like a copilot
- No dashboard overview at a glance
- More complex to build
- Less useful without AI — needs manual nav fallback
